# Risk Agent API

An AI-powered document analysis service that generates risk scores, summaries, and approval recommendations for financial and legal documents. The agent combines deterministic risk scoring with an LLM-generated summary via the Anthropic API.

---

## Features

- **Risk Scoring** — composite score (0.0–1.0) built from four weighted components: validation, completeness, anomaly, and schema risk
- **Recommendation Engine** — automatic `approve`, `review`, or `reject` decisions based on thresholds
- **AI Summaries** — 2–3 sentence professional summaries generated by Claude (falls back gracefully if no API key is set)
- **Batch Processing** — analyze up to 20 documents concurrently in a single request
- **Interactive Docs** — auto-generated Swagger UI at `/docs` and ReDoc at `/redoc`

## Running the Server

```bash
# default: http://127.0.0.1:8000
python app.py

# or via uv
uv run python app.py
```

Environment variables:

| Variable            | Default     | Description                                      |
|---------------------|-------------|--------------------------------------------------|
| `PORT`              | `8000`      | Port the server listens on                       |
| `UVICORN_RELOAD`    | `0`         | Set to `1` to enable hot-reload (development)    |
| `ANTHROPIC_API_KEY` | *(not set)* | Enables AI-generated summaries via Claude        |

If `ANTHROPIC_API_KEY` is not set, the API still works — summaries are replaced with a placeholder message.

## API Reference

### `GET /health`
Liveness probe.

```json
{
  "status": "ok",
  "agent": "risk-agent",
  "version": "1.0.0",
  "timestamp": "2025-01-15T12:00:00Z"
}
```

---

### `POST /api/v1/analyze`
Analyze a single document and return a full risk assessment.

**Request body:**

```json
{
  "document_id": "DOC001",
  "standardized_data": {
    "document_type": "invoice",
    "issuer": "Acme Corp",
    "amount": 15000.00,
    "currency": "USD",
    "issue_date": "2024-01-15",
    "expiry_date": "2024-04-15",
    "counterparty": "Globex Ltd",
    "jurisdiction": "US",
    "metadata": {"po_number": "PO-9981"}
  },
  "validation_result": {
    "is_valid": true,
    "missing_fields": [],
    "anomalies": [],
    "schema_errors": [],
    "completeness_score": 0.97
  }
}
```

**Response:**

```json
{
  "document_id": "DOC001",
  "summary": "The invoice from Acme Corp...",
  "risk_score": 0.0075,
  "risk_breakdown": {
    "validation_risk": 0.0,
    "completeness_risk": 0.03,
    "anomaly_risk": 0.0,
    "schema_risk": 0.0,
    "overall_risk": 0.0075
  },
  "recommendation": "approve",
  "confidence": 0.88,
  "reasoning": "Risk score of 0.0075 places this document in the 'approve' category...",
  "flags": [],
  "processed_at": "2025-01-15T12:00:00Z"
}
```

---

### `POST /api/v1/batch`
Process up to **20 documents** concurrently. The request body is an array of the same objects accepted by `/analyze`. Results are returned in the same order as the input.

## Risk Scoring

The composite risk score is a weighted sum of four components:

| Component          | Weight | Description                                         |
|--------------------|--------|-----------------------------------------------------|
| `validation_risk`  | 30%    | Binary penalty for invalid documents + missing fields |
| `completeness_risk`| 25%    | Inverse of the completeness score                   |
| `anomaly_risk`     | 25%    | Sigmoid-shaped penalty based on number of anomalies |
| `schema_risk`      | 20%    | Linear penalty per schema error (capped at 1.0)     |

**Recommendation thresholds:**

| Score Range | Recommendation |
|-------------|----------------|
| 0.00 – 0.25 | `approve`      |
| 0.26 – 0.55 | `review`       |
| 0.56 – 1.00 | `reject`       |
